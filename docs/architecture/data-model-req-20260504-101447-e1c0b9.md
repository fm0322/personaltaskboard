# Data Model — Personal Task Board

## 1) Persistence Choice
- Database: **SQLite**
- File: `taskboard.db`
- ORM: **Entity Framework Core**
- Migration strategy: **auto-apply on startup** (`Database.Migrate()`), then seed defaults.

## 2) Entity Model (Fully Typed)

SQLite storage mapping guidance:
- `Guid` -> `TEXT` (canonical string)
- `string` -> `TEXT`
- `int` / `int?` -> `INTEGER`
- `DateOnly` -> `TEXT` (`yyyy-MM-dd`)
- `DateTimeOffset` -> `TEXT` (ISO-8601)

## Board
- `Id: Guid` (PK)
- `Name: string` (required, max 100)
- `CreatedAt: DateTimeOffset` (required)
- `UpdatedAt: DateTimeOffset` (required)

Constraints:
- Single default board in current scope (one active board).

Relationships:
- `Board (1) -> (many) Column`

## Column
- `Id: Guid` (PK)
- `BoardId: Guid` (FK -> Board.Id, required)
- `Name: string` (required, max 80)
- `DisplayOrder: int` (required, zero-based recommended)
- `WipLimit: int?` (optional, must be >= 1 when present)
- `CreatedAt: DateTimeOffset` (required)
- `UpdatedAt: DateTimeOffset` (required)

Constraints:
- Unique `(BoardId, Name)` (case-insensitive collation decision left to implementation).
- Unique `(BoardId, DisplayOrder)` for stable column ordering.

Relationships:
- `Column (1) -> (many) TaskItem`

## TaskItem
- `Id: Guid` (PK)
- `ColumnId: Guid` (FK -> Column.Id, required)
- `Title: string` (required, max 200)
- `Description: string?` (optional, max 4000)
- `DueDate: DateOnly?` (optional)
- `Priority: Priority` (required enum)
- `AssigneeLabel: string?` (optional, max 60)
- `SortOrder: int` (required, zero-based recommended per column)
- `CreatedAt: DateTimeOffset` (required)
- `UpdatedAt: DateTimeOffset` (required)

Constraints:
- Unique `(ColumnId, SortOrder)` for deterministic card order in each column.
- Index on `(ColumnId, SortOrder)`.
- Index on `Priority`.
- Index on `Title` (supports contains search with SQLite limitations).
- Optional composite index `(ColumnId, Priority)` for common filters.

## Priority Enum
- `Low = 0`
- `Medium = 1`
- `High = 2`
- `Urgent = 3`

## 4) DbContext Design
`TaskBoardDbContext` includes:
- `DbSet<Board> Boards`
- `DbSet<Column> Columns`
- `DbSet<TaskItem> TaskItems`

Model configuration expectations:
- Required fields and max lengths per entity definitions above.
- Enum `Priority` stored as integer.
- FK delete behavior:
  - Board delete cascades to Columns.
  - Column delete cascades to TaskItems.
- Unique indexes:
  - `Column(BoardId, DisplayOrder)`
  - `TaskItem(ColumnId, SortOrder)`
  - `Column(BoardId, Name)`

## 5) Migration + Startup Flow
Startup sequence:
1. Build host.
2. Open scoped service provider.
3. Resolve `TaskBoardDbContext`.
4. Execute `Database.Migrate()`.
5. Seed default `Board` + default columns (`To Do`, `In Progress`, `Done`) if missing.
6. Start web app.

Benefits:
- New developer onboarding requires only `dotnet run`.
- DB schema and defaults are auto-initialized consistently.
